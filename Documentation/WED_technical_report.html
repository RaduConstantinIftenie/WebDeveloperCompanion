Abstract

This project aims to create an ontology based on the linked data principles providing modern solutions for querying programming data related to web development. It provides multiple methods of interactions such as UI/UX in react and a SPARQL endpoint, facilitating data ingestion and exporting for any kind of systems.

Introduction

In the current age of technology, the web technologies advance at an exponential pace. Members of the web developing community, regardless of technologies or seniority, constantly seek to stay up-to-date to the latest developments, features and achievements in the field. However, in the late years, it has become increasingly difficult to keep up with the big number of new technologies and advancements, most of the time leading to inaccurate, outdated or incomplete data.

Our solution aims to help developers and web technologies passionates stay up-to-date with the latest technologies by created service-oriented system that manages and models the widely available technical context. Furthermore, by using the linked data principles it aims to connect multiple sources of information and validate, enrich and standardize the provided data. Moreover, by providing both a modern UI and a SPARQL endpoint we enable both user and software interaction.

Architecture

Front-end

Front-end's architecture aims to provide a modern user experience by using a top of the line UI. The project is written using React and using the best practices regarding folder structure and component composition and injection.

Back-end

Back-end's architecture follows a serverless approach using JAVA and Node.js and provides a highly scalable, available solution for our application. The projects have n-tier internal architecture for the JAVA projects and pseudo-DDD for Node.js one. The scrapper will be written in python.

Infrastructure

The infrastructure is bound to the AWS cloud provider and offers a serverless approach. Furthermore, we use several components that assure the Security and privacy of our solution.

Data Models

The application's main goal is to handle data in a standardized way, including both user inserted and external data. The user data is mainly handled by Keycloack and will be stored in its internal state and storage system, while the external data will be validated, cleaned and stored in AWS Neptune as RDF data. The external data will be retrieved from multiple sources using a python service that will automatically scrape webpages. Additionally, the data will be cleaned, formatted and grouped into different categories.

Conclusion

As we are looking ahead to the implementation of the project, at this point in time, we conclude by our wish to build a robust, highly scalable, highly available service with a great user experience that will be used by both users and software alike.
